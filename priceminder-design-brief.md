# PriceMinder — Design Asset Request for JRS Digital Website

## Context
The jrsdigital.net site currently reuses 4 real app screenshots across the homepage, Work page, and PriceMinder product page: Dashboard/Overview, Library (subscription list), Offers (NBN/mobile comparison), and Edit Subscription (promo tracking). That's thin — three pages share the same handful of images, and a couple of the site's core claims aren't visually backed by any screen yet.

## Existing app design system (for reference/consistency)
- Background: near-black charcoal (`#0B0B11`–`#14141C` range)
- Primary accent: warm gold (`~#C8A96E`), used for calendar highlights, key numbers, primary CTAs on some screens
- Secondary accent: purple/violet gradient (`~#6C63FF → #8B7FFF`), used for the active bottom-nav pill and some buttons/toggles
- Status colors: green for "renews in X days," red for delete/destructive actions
- Typography: serif display face (Playfair Display) for big numeric headers like "$370.91" or "11 subscriptions"; clean sans-serif (Plus Jakarta Sans / Space Grotesk) for UI text
- Bottom nav: 4 tabs — Library, Dashboard, Offers, Settings

New screens should match this existing in-app visual language — **not** the marketing site's newer slate-blue "wide theme" (that palette is for the website chrome only, not the app itself).

## New screens needed
1. **Price-alert / notification screen** — visualizes the site's core promise ("catch every price hike before it costs you"). An in-app alert or banner showing a detected price rise or an expiring promo with a clear call to action. This is the one claim on the site with no matching visual right now.
2. **Settings / privacy screen** — reinforces "no accounts, no analytics, on-device only." Toggle rows, an "Export data" action, no login/account section visible.
3. **Onboarding or empty-library state** — a first-run screen with no data yet, backing up the "no sign-up required" claim. Useful as an alternate hero image.
4. **App icon** — square source at 1024×1024, needs to read clearly as a small favicon too (currently using a placeholder mark). This gets used both on the Play Store listing and as the site favicon.
5. **Social share image** — 1200×630, for link previews when the site is shared (Slack, X, iMessage, etc.). This one should use the **website's** palette, not the app's: background `#EEF1F1`, ink `#1B2224`, accent `#5C7C93`. Currently a plain placeholder generated by Claude — needs a real designed version.

## Nice to have
Re-export the 4 existing screens (Dashboard, Library, Offers, Edit Subscription) at 2x resolution. The current files are 750×1608px and look slightly soft when scaled up in the new wider homepage layout.

## Technical specs
- Portrait phone crop, consistent aspect ratio (~1:2.14, matching current screens — status bar to bottom, no home-indicator cropping artifacts)
- Export at 2x (1500×3216) or 3x if available
- Format: PNG or WebP
- Suggested filenames (kebab-case, matches existing convention): `priceminder-alert.webp`, `priceminder-settings.webp`, `priceminder-onboarding.webp`, `app-icon-1024.png`, `og-share-image.png`

## 3D hero render — image-gen prompt

For a static, photorealistic "studio-lit" 3D render (e.g. a hero image, App Store feature graphic, or press image), tuned to the actual brand — not generic gradients:

> A single smartphone floating at a slight three-quarter angle against a soft, dark charcoal-to-black studio backdrop, shot like a premium tech product render. The phone body is a matte-to-satin dark graphite metal (near-black, #1B1B20-ish) with a thin polished bevel edge, softly rounded corners, and a single small punch-hole front camera — no visible brand logos. The screen displays a dark-mode finance app dashboard: a donut chart, a calendar grid, and a large dollar figure in a serif typeface, lit as if the display is genuinely emitting light. A soft violet glow (#5852ff) emanates from the screen, bleeding gently onto the phone's edges and casting a subtle colored reflection below it on an unseen glossy surface. Three-point studio lighting: a bright key light from the upper left, a soft fill from the right, and a cool rim light from behind to separate the phone from the background. Shallow depth of field, subtle floor reflection, 50mm product-photography lens look, high detail, no text or UI elements outside the phone screen, no hands, no other objects in frame.

Notes for whoever runs this:
- `#5852ff` is the same violet used for the live 3D scene on the site (`assets/hero-3d.js`) and is close to the app's real in-app button/accent color — keep it consistent if you generate more than one render.
- Swap "finance app dashboard" details for whichever screen you want to feature (e.g. the Offers comparison or Library list) by describing that screen's actual layout instead.
- If the tool supports negative prompts, exclude: extra phones, warped text, distorted UI, cartoon/toy proportions, bright white plastic body (this is exactly the "kids phone" look we moved away from in the CSS mockups).
